# playbooks

YAML engagement playbooks. Each file defines a structured attack plan composed
of Atomic Red Team test steps mapped to MITRE ATT&CK techniques. Playbooks are
consumed by `modules/playbook_engine.py` and surfaced via the `apt_playbook`
CLI command and the `lazyown_playbook_generate` / `lazyown_playbook_run` MCP
tools.

## File types

| Pattern | Type | Description |
|---------|------|-------------|
| `apt_*.yaml` | APT simulation | Full kill-chain playbook based on a documented threat actor. Techniques sourced from published CTI reports. |
| `attack_plan*.yaml` | Generated plan | Auto-generated by `apt_playbook run <apt_name>`. Tailored to the current target OS and discovered services. |
| `ExamplePlaybook.yaml` | Template | Reference template for writing new playbooks by hand. |

## Playbook YAML structure

```yaml
apt_name: apt29
description: APT29 simulated attack chain
phases:
  - phase: initial_access
    techniques:
      - id: T1566.001
        name: Spearphishing Attachment
        atomic_id: T1566.001-1
        command: "python3 modules/phishing_gen.py"
        platform: linux
  - phase: persistence
    techniques:
      - id: T1053.003
        name: Cron Job
        atomic_id: T1053.003-2
        command: "crontab -l && echo '* * * * * /tmp/.svc' | crontab -"
        platform: linux
```

## CLI usage

```
(LazyOwn) > apt_playbook list
(LazyOwn) > apt_playbook info apt29
(LazyOwn) > apt_playbook validate apt29
(LazyOwn) > apt_playbook run apt29
(LazyOwn) > apt_playbook report apt29
```

`apt_playbook run` generates `sessions/attack_plan.yaml` tailored to the
current `rhost` OS. Review it before execution.

## MCP usage

```
lazyown_playbook_generate(apt_name="apt29", platform="linux")
lazyown_playbook_run(playbook_path="sessions/attack_plan.yaml")
```
