# sessions

Authoritative campaign state directory. Every component of the framework
reads from and writes to this directory. Never delete its contents without
explicit operator confirmation — this is the engagement record.

## Key files

| File / Pattern | Producer | Consumer | Notes |
|----------------|----------|----------|-------|
| `scan_<rhost>.nmap` | `do_lazynmap` | autonomous_daemon, pwntomate, FactStore | Full nmap output. Check before rescanning. |
| `scan_<rhost>.nmap.xml` | `do_lazynmap` | pwntomate (XML parser), obs_parser | Machine-readable port/service data. |
| `vulns_<rhost>.nmap` | `do_lazynmap` (vuln scripts) | reactive_engine | Vulnerability script output. |
| `world_model.json` | autonomous_daemon | session_state, recommend_next, dashboard | Current phase, hosts, services, credentials, objectives. |
| `tasks.json` | campaign_tasks, MCP | sitrep, dashboard, tasks_cleanup | Active task list. |
| `objectives.jsonl` | inject_objective | autonomous_daemon | Ordered objective queue. One JSON object per line. |
| `LazyOwn_session_report.csv` | every `run_command` call | timeline, threat_model, inline hints | Full command history with timestamps. |
| `sessionLazyOwn.json` | shell shutdown, shift handoff | sitrep, c2_notes | Shift summary exported on exit. |
| `credentials*.txt` | reactive_engine, do_responder, hashcat | every later-phase command | Cleartext credentials. Checked by autonomous selectors before auth attempts. |
| `hash*.txt` | do_responder, do_secretsdump | john, hashcat | NTLM hashes and other captured credential material. |
| `os.json` | do_ping, beacon | every OS-aware selector | Ground-truth OS for the current target. |
| `events.jsonl` | event_engine | poll_events MCP tool | Structured engagement events. |
| `autonomous_events.jsonl` | autonomous_daemon | poll_events MCP tool | Daemon-specific events (phase change, reward, error). |
| `campaign_lessons.jsonl` | EpisodeReflectionEngine | next campaign bootstrap | Lessons extracted after each engagement episode. |
| `policy_facts.json` | policy engine | dashboard, auto_loop | Reward and detection scores per tactic. |
| `soul.md` | operator via `lazyown_soul` | every LLM agent | Persona and hard-stop policy for all AI agents. |
| `access.log` | lazyc2 logging | operator review | HTTP access log for the C2 server. |
| `engagement_state.json` | engagement_hooks | ops_commands | Tracks engagement start time and objectives. |
| `blacksandbeacon` | `blacksandbeacon` addon | served by C2 file endpoint | Compiled Linux C beacon binary. |
| `bof_loader` | `blacksandbeacon_bof` addon | served by C2 file endpoint | Linux BOF loader binary. |
| `keyword_fallback_index.json` | rag_index when ChromaDB absent | rag_query | Keyword-based RAG fallback index. |

## Subdirectories

| Directory | Contents |
|-----------|---------|
| `<ip>/<port>/<tool>/` | Pwntomate tool output. Created automatically for each discovered service. |
| `logs/` | Per-command log files: `command_<tool>output<domain>.txt`. |
| `c2_profiles/` | Generated malleable C2 profile variants. |
| `captured_images/` | Images captured by the decoy landing page (webcam stills). |
| `phishing/` | Phishing campaign tracking data: click logs, form submissions. |
| `reports/` | Generated PDF and HTML engagement reports. |
| `hive/` | Hive-mind drone output and ChromaDB fallback data. |
| `implant/` | Implant configuration files and staging artefacts. |
| `uploads/` | Files uploaded by operators through the web UI. |
| `temp_uploads/` | Temporary staging area cleared after processing. |
| `cve_cache/` | NVD CVE JSON cache. Populated by `do_nvddb`. |
| `ai_model/` | Cached LLM responses and fine-tuning data. |
| `win/` | Windows-specific artefacts (shellcode, stagers). |
| `lin/` | Linux-specific artefacts (ELF payloads, BOF objects). |
| `c/` | C source generated by the framework (custom stubs, payloads). |

## Before running any tool

1. `ls sessions/` to see what already exists.
2. If `scan_<rhost>.nmap` is present and recent, run `facts_show` instead of
   `lazynmap`. Rescanning wastes 30+ minutes and alerts defenders.
3. If `credentials*.txt` is non-empty, check it before attempting any
   authentication — the creds are already there.

## Backup and cleanup

Sessions are excluded from git via `.gitignore`. To archive an engagement:

```bash
tar -czf engagement_$(date +%Y%m%d).tar.gz sessions/
```

To start a fresh engagement without deleting the archive:

```bash
mv sessions/ sessions_$(date +%Y%m%d)/
mkdir sessions/
```
