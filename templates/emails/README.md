# templates/emails/

YAML email payload templates used by the LazyOwn phishing engine to compose
outbound messages. Consumed by the AI composer in `modules/phishing*.py`
(and the multivector orchestrator) when an operator picks a template in
`templates/phishing/new_campaign.html` or `create_multivector_campaign.html`.

## Files

| File pattern | Origin | Notes |
|--------------|--------|-------|
| `bank_alert.yaml` | Hand-authored baseline | Generic banking-alert pretext; safe starting point. |
| `ai_template_<unix_ts>.yaml` | Auto-generated by the AI composer | Produced when an operator clicks "Generate variant". Filename carries the creation epoch so collisions are impossible. |

## YAML schema

```yaml
name: "Bank Alert"                    # operator-facing label
subject: "URGENT: verify account"     # email subject
sender_name: "Security Team"          # display name
sender_email: "no-reply@bank.tld"     # spoofed From (rendered into the SMTP envelope)
body_html: |                          # Jinja2 string rendered against the campaign context
  <p>Dear {{ target_name }},</p>
  <p>Click <a href="{{ landing_url }}">here</a> to verify.</p>
body_text: |                          # plain-text fallback (auto-stripped from html when omitted)
  Dear {{ target_name }}, ...
tags: ["banking", "urgency"]          # used for filtering in campaigns.html
```

`landing_url`, `target_name`, `target_email` are always injected by the
campaign runtime. Any additional placeholder must be declared in the
campaign payload before rendering.

## Adding a template

- Hand-authored files: pick a stable, descriptive lowercase snake-case stem (`linkedin_invite.yaml`, `crypto_giveaway.yaml`).
- Never commit a template that targets a real organisation or carries real PII.
- Validate YAML with `python -c 'import yaml,sys; yaml.safe_load(open(sys.argv[1]))' templates/emails/<file>.yaml` before committing.
- AI-generated `ai_template_*.yaml` files are gitignored by convention — keep them out of source control.
