# modules/rootkit

Linux kernel module rootkit research code. Demonstrates kernel-level hiding
techniques for authorized red team engagements and security research. Requires
a kernel development environment and root access to load. For authorized use
only on systems you own or have explicit written permission to test.

## Files

| File | Description |
|------|-------------|
| `mr.c` | Minimal rootkit kernel module. Hides files with a configurable prefix, hides the module itself from `lsmod`, and intercepts the `kill` syscall to accept control commands. |
| `mrhyde2.c` | Extended variant with process hiding, network socket hiding, and a covert communication channel via `procfs`. |
| `Module.symvers` | Kernel symbol version file generated during build. Required for out-of-tree module compilation. |
| `modules.order` | Module dependency order file. Generated by `make`. |
| `__init__.py` | Python wrapper that builds and loads the kernel module via `subprocess`. Called by the CLI `do_rootkit` command. |

## Building

```bash
cd modules/rootkit
make -C /lib/modules/$(uname -r)/build M=$(pwd) modules
```

## Loading and unloading

```bash
# Load
sudo insmod mr.ko

# Verify loaded (before hiding)
lsmod | grep mr

# Unload
sudo rmmod mr
```

## Usage from the CLI

```
(LazyOwn) > rootkit install   # build and insmod
(LazyOwn) > rootkit hide      # activate file/process hiding
(LazyOwn) > rootkit remove    # rmmod and clean up
```

## Kernel compatibility

The module is developed against kernel 6.x. Compilation against older kernels
may require source adjustments due to syscall table and procfs API changes.
Always test in a VM before deploying in an engagement.
