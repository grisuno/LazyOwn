# docs

Published documentation site for LazyOwn. The HTML files here are the static
site served at `https://grisuno.github.io/LazyOwn/` via GitHub Pages.

## Files

| File | Description |
|------|-------------|
| `index.html` | Main documentation page. Auto-generated from `README.md` by `DEPLOY.sh`. Do not edit by hand — edits will be overwritten on the next deploy. |
| `README.html` | HTML version of the root `README.md`. |
| `COMMANDS.html` | HTML version of `COMMANDS.md` — exhaustive CLI command reference. |
| `UTILS.html` | HTML version of `UTILS.md` — auto-generated `utils.py` API reference. |
| `LAZYOWN.html` | HTML version of `skills/lazyown.md` — full MCP tool playbook. |
| `CHANGELOG.html` | HTML version of `CHANGELOG.md`. |
| `SECURITY_PLAN.md` | Security hardening plan and known mitigations. |
| `WORLD_CLASS_PLAN.md` | Long-term architecture and feature roadmap. |
| `style.css` | Stylesheet for all documentation pages. |
| `template.html` | Base HTML template used by the DEPLOY pipeline to generate the static pages. |
| `conf.py` | Sphinx configuration (used when building the Sphinx variant of the docs). |
| `index.rst` | Sphinx root document. |
| `CNAME` | Custom domain record for GitHub Pages. |
| `favicon.ico` | Browser tab icon. |

## Updating the docs

The HTML files are auto-generated by `DEPLOY.sh`. To preview locally:

```bash
python -m http.server 8080 --directory docs/
# Open http://localhost:8080
```

To force a docs-only rebuild without creating a release commit:

```bash
python readmeneitor.py lazyown.py
python readmeneitor.py utils.py
bash scripts/update_index.sh
```

## Editing prose documentation

Edit the source Markdown files at the repo root (`README.md`, `COMMANDS.md`,
`UTILS.md`, `CHANGELOG.md`). The HTML in `docs/` is always derived from those
files — editing HTML directly is wasted work.
